# Reusable Components

Identify duplicate UI patterns and refactor into reusable components.

## Analysis
1. Scan for repeated UI patterns (buttons, cards, inputs, etc.)
2. Identify variations in styling/behavior
3. Find consolidation opportunities

## Refactoring Guidelines
- Create flexible components with props for variations
- Use consistent naming conventions
- Add TypeScript types/interfaces
- Consider composition over configuration
- Keep components focused and single-purpose

## Frameworks
Adapt patterns for: React, Angular, Vue, Next.js, Svelte as needed
