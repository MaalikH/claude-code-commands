# Architecture

Explain the application structure and data flow.

## Explain
- Overall system architecture
- How data flows from user action to service and back
- Key components and their responsibilities
- Service boundaries and communication
- Database and state management
- External integrations

## Format
Present like explaining a diagram:
- Start with high-level overview
- Trace the path of a typical request
- Explain each layer and its role
- Show how components connect
